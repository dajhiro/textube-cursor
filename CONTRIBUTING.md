# 기여 가이드

## 개발 환경 설정

1. 저장소 클론
2. 의존성 설치: `npm install`
3. 데이터베이스 설정 (Docker Compose 사용 권장)
4. 환경 변수 설정
5. 개발 서버 실행

## 코드 스타일

- TypeScript 사용
- ESLint 규칙 준수
- Prettier 포맷팅 사용 (예정)

## 커밋 메시지

- feat: 새 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 포맷팅
- refactor: 코드 리팩토링
- test: 테스트 추가
- chore: 빌드 설정 등

## Pull Request

1. 기능 브랜치 생성
2. 변경사항 커밋
3. 테스트 실행
4. Pull Request 생성

## 이슈 리포트

버그나 기능 요청은 GitHub Issues를 사용해주세요.
